# Компьютер и хранение

## Требования к процессору (CPU)

**DAW** — вычислительно интенсивное приложение. Аудио требует **реального времени** без пропусков: даже 1 пропущенный буфер = слышимый артефакт (кликающий звук).

### Ключевые параметры CPU

**Однопоточная производительность** важнее количества ядер. Большинство плагинов работают в один поток на канал. Высокочастотный процессор с меньшим числом ядер часто лучше для аудио.

**Количество ядер**: важно для больших проектов с параллельной обработкой. Современные DAW хорошо используют 8+ ядер.

### Современные рекомендации (2025)

| Платформа | Рекомендуемые CPU |
|----------|-----------------|
| **Windows** | Intel Core i7/i9 14-го поколения; AMD Ryzen 9 7000-серия |
| **Mac** | Apple M2/M3/M4 — исключительная производительность для аудио |
| **Бюджетно** | Intel i5/i7 12–13 поколения, AMD Ryzen 5/7 5000 |

**Apple Silicon (M1/M2/M3/M4)** — революционная производительность на ватт. Logic Pro и большинство DAW оптимизированы под ARM. Рекомендовано для покупки нового оборудования.

### Тест производительности: DPPM

Для сравнения CPU в DAW-контексте используют **DPC Latency Checker** (Windows) или **DAWBench** — тест на количество плагинов при фиксированном буфере.

## Оперативная память (RAM)

**Минимум:** 16 ГБ
**Рекомендуется:** 32 ГБ
**Для профессиональных проектов (оркестр, много плагинов с библиотеками):** 64+ ГБ

Большинство виртуальных инструментов загружают семплы в RAM:
- Легкие плагины: 500 МБ–2 ГБ
- Оркестровые библиотеки (EastWest, Berlin Orchestra): 8–32 ГБ и более

## Жёсткие диски и SSD

### SSD для аудио-проекта — обязательно

**HDD** (традиционный диск) имеет время доступа 5–15 мс — слишком медленно для многодорожечного аудио при работе с буфером 64–128 семплов.

**SSD** — время доступа < 0,1 мс. Обязателен для операционной системы и проектов.

**NVMe SSD (PCIe)** — ещё быстрее. Рекомендуется для системного диска и рабочих проектов.

### Рекомендуемая структура дисков

| Диск | Тип | Назначение |
|------|-----|----------|
| **Диск C / System** | NVMe SSD 500 ГБ–1 ТБ | ОС, DAW, плагины |
| **Диск D / Projects** | SATA SSD 1–2 ТБ | Текущие аудиопроекты |
| **Диск E / Samples** | SATA SSD или HDD 2–4 ТБ | Библиотеки семплов |
| **Внешний диск / Backup** | HDD 4+ ТБ | Резервные копии |

**Почему разделять системный диск и проекты?**
- Когда ОС активно обращается к системному диску (обновления, кэш), это не мешает чтению аудио-файлов
- Проще делать резервные копии проектов отдельно от системы

## Стратегия резервного копирования (Backup)

Потеря записанного материала — одна из самых болезненных ситуаций в музыке. Правило **3-2-1:**

```
3 копии данных
2 разных носителя
1 копия вне дома (offsite или облако)
```

### Практическая стратегия

**Уровень 1: Локальный backup**
- Внешний диск (Time Machine на Mac / Windows Backup)
- Автоматически каждый день
- Хранить минимум 2 недели истории (чтобы можно было откатиться к более старой версии)

**Уровень 2: Облачный backup**
| Сервис | Стоимость | Особенности |
|-------|---------|------------|
| **Backblaze** | $9/мес — неограниченно | Лучшее соотношение цена/объём |
| **iCloud (Mac)** | 50–200 ГБ бесплатно | Интегрирован в macOS |
| **Google Drive / Dropbox** | Ограниченный бесплатный объём | Удобно для небольших проектов |
| **Synology NAS + Hyper Backup** | Разовая инвестиция | Профессиональное NAS-решение |

**Уровень 3: Архивирование завершённых проектов**
- Завершённый проект → архив ZIP с аудиофайлами, миксами и мастерами → внешний диск

### Что включать в backup проекта

- Файл проекта DAW (.als, .logic, .ptx и т.д.)
- Все аудиофайлы проекта (не просто ссылки)
- Пресеты плагинов, использованные в проекте
- Финальные миксы и мастера в WAV 24 бит

## Оптимизация системы для аудио

### Windows

- Отключить Hyper-Threading? Зависит от DAW: Ableton выигрывает при отключении, другие — нет
- **Timer resolution:** использовать Bitsum Process Lasso или HPET настройку
- **BIOS:** отключить C-states и SpeedStep для снижения DPC-latency
- Аудиодрайвер: использовать **ASIO** (не WDM/MME)

### macOS

- Оптимальная платформа из коробки для аудио
- Не нужно отдельно настраивать — Darwin kernel имеет низкую задержку
- **Logic Pro** максимально оптимизирован для Apple Silicon

### Общие правила

- Не устанавливать лишнее ПО на «студийный» компьютер
- Отключить антивирусную проверку папок с аудиопроектами (или добавить в исключения)
- Wi-Fi-адаптер во время записи — лучше отключить (может вызывать интерференцию)

> **Золотое правило:** студийный компьютер — специализированное устройство. Не используйте его для игр, сёрфинга в интернете и повседневных задач. Чем меньше посторонних процессов — тем стабильнее работает аудио.
