# Базовый workflow в домашней студии

## Структура DAW-проекта

**DAW** (Digital Audio Workstation) — центр домашней студии. Всё — запись, редактирование, сведение, мастеринг — происходит здесь.

Популярные DAW:

| DAW | Платформа | Сильные стороны |
|-----|---------|----------------|
| **Ableton Live** | Mac/Win | Live-выступления, электронная музыка, loop-based |
| **Logic Pro** | Mac only | Встроенные плагины, рок/поп, отличная цена |
| **Pro Tools** | Mac/Win | Индустриальный стандарт для записи и постпродакшена |
| **Cubase** | Mac/Win | Нотация, MIDI, film scoring |
| **FL Studio** | Mac/Win (Win родной) | Биты, trap, EDM |
| **Reaper** | Mac/Win/Linux | Гибкий, дешёвый, кастомизируемый |
| **Bitwig** | Mac/Win/Linux | Модульный, эксперименты |

## Создание проекта: хорошие привычки с начала

### Структура папок проекта

```
/Проекты/
  Название_трека_v1/
    ├── Проект.als (или .logic, .cpr)
    ├── Audio/
    │   ├── Recorded/        ← все записанные файлы
    │   └── Bounced/         ← экспортированные стемы и миксы
    ├── Midi/                ← MIDI-клипы (если хранятся отдельно)
    ├── Samples/             ← сэмплы, специфичные для проекта
    └── Refs/                ← reference-треки
```

**Принцип:** никогда не полагайтесь на «абсолютные пути» к сэмплам. Всегда собирайте все файлы проекта в одну папку (consolidate project).

### Именование треков

Хороший трек с первых дней работы:
- Вокал: `VOC_Lead`, `VOC_BV_L`, `VOC_BV_R`
- Инструменты: `GTR_Rhythm_L`, `GTR_Lead`, `BASS_DI`
- Барабаны: `DRUMS_Kick`, `DRUMS_Snare`, `DRUMS_OH_L/R`
- Группы: `GRP_Drums`, `GRP_Guitars`, `GRP_Vox`

Непонятные имена «Audio 1», «MIDI 5» — потеря времени при сведении.

## Signal Routing (маршрутизация сигнала)

### Шины (Buses / Groups)

**Групповая шина** объединяет несколько треков → позволяет управлять ими вместе и применять обработку ко всей группе.

```
DRUMS_Kick   ┐
DRUMS_Snare  ├→ GRP_Drums Bus → Компрессор → Master
DRUMS_OH_L/R ┘
```

**Зачем это нужно:**
- Единый контроль громкости группы
- Параллельное сжатие (parallel compression)
- Групповой EQ или сатурация
- Экономия CPU (один эффект на группу вместо нескольких одинаковых)

### Aux-шины для эффектов

**Send/Return** (FX Bus) — эффективный способ использования реверба и delay:

```
Вокал → [Send 30%] → AUX Reverb Bus → [Reverb Plugin] → Master
Гитара → [Send 15%] → AUX Reverb Bus (тот же)
```

**Преимущества:**
- Один плагин реверба для нескольких треков (экономия CPU)
- Единое пространство — звук «клеится» вместе
- Простота изменения общего характера реверба

### Master шина

Все треки и группы в итоге попадают на **Master Bus** (Main Mix). На нём обычно:
- **EQ** — небольшая коррекция общего баланса
- **Компрессор-клей** (glue compressor) — мягкое, незаметное сжатие всего микса
- **Лимитер** — для конкурентного уровня (при мастеринге)
- **Meter** — контроль уровней (LUFS, True Peak)

## Шаблоны (Templates)

**Шаблон** — заранее подготовленный проект со всеми стандартными треками, шинами, роутингом и настройками. Цель — не начинать с нуля каждый раз.

### Что включать в шаблон

- Группы для стандартных инструментов (барабаны, бас, гитары, вокал)
- AUX-шины для реверба (комната, пластина, большой зал), delay (eighth-note, quarter-note)
- Master bus с компрессором и лимитером
- Reference track шина
- Loudness meter на мастере
- Папки/группы свёрнуты — чистый вид

### Создание шаблона

В большинстве DAW: **File → Save As Template** (или аналог). При создании нового проекта выбрать шаблон.

**Совет:** создайте несколько шаблонов под разные задачи (рок-запись, электронная музыка, подкаст).

## Общий workflow: от идеи до финала

### 1. Препродакшен (Preproduction)

- Записать демо голосом или телефоном
- Определить BPM и тональность
- Выбрать reference-треки

### 2. Запись (Recording)

- Создать проект из шаблона
- Записать базовые треки (барабаны или клик + бас-гитара)
- Наложить остальные инструменты
- Записать вокал последним (обычно)

**Правило при записи:** не сводить во время записи. Запись — это отдельный этап.

### 3. Редактирование (Editing)

- Чистка: убрать шум, щелчки, лишние пустые места
- Квантизация ударных (если нужно)
- Тюнинг вокала (Melodyne, Auto-Tune)
- Выравнивание питча и тайминга

### 4. Сведение (Mixing)

- Статичный баланс уровней
- Панорамирование
- EQ
- Компрессия
- Эффекты (реверб, delay)
- Автоматизация

### 5. Мастеринг (Mastering)

- Финальный EQ и компрессия
- Лимитинг до целевого уровня (−14 LUFS для стриминга, −9 LUFS для клубного)
- Экспорт финала

## Контроль версий проектов

Сохраняйте инкрементальные версии:
```
Track_v01.als
Track_v02.als
Track_mix_v01.als
Track_mix_v02_bass_fixed.als
Track_final.als
Track_MASTER.als
```

Никогда не перезаписывайте единственный файл проекта — всегда сохраняйте новую версию.

> **Главное правило домашней студии:** workflow — это привычки. Хорошие привычки (именование, структура, регулярный backup, шаблоны) экономят часы работы и предотвращают потерю данных. Инвестируйте время в правильную организацию с первого проекта.
