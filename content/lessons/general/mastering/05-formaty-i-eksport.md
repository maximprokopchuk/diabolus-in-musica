# Форматы и экспорт

## Аудиоформаты: основы

### Sample Rate (частота дискретизации)

**Sample Rate** — количество «снимков» (сэмплов) сигнала в секунду.

| Sample Rate | Применение |
|------------|-----------|
| 44.1 кГц | Стандарт CD и стриминга |
| 48 кГц | Видео и кино (стандарт AES) |
| 88.2 / 96 кГц | Студийная запись, мастеринг |
| 192 кГц | Hi-Res Audio |

**Теорема Найквиста:** для точного воспроизведения частоты F нужен sample rate ≥ 2F. При 44.1 кГц воспроизводится до 22.05 кГц — выше порога слышимости человека (~20 кГц).

**Для финального экспорта:**
- Стриминг: 44.1 кГц
- Видео: 48 кГц
- Оставьте исходный sample rate проекта

### Bit Depth (битовая глубина)

**Bit Depth** определяет **динамический диапазон** записи.

| Bit Depth | Динамический диапазон | Применение |
|----------|----------------------|-----------|
| 16 bit | ~96 дБ | CD, финальный экспорт |
| 24 bit | ~144 дБ | Студийная работа, мастер-файлы |
| 32 bit float | Неограниченный | Внутри DAW, без клиппинга |

**Правило:** записывайте и работайте в **24 bit**. Финальный мастер для CD/стриминга — **16 bit** (с dithering).

## Dithering (дизеринг)

### Зачем нужен дизеринг

При конвертации из 24 bit в 16 bit теряются 8 бит информации. Это вносит **quantization noise** (шум квантования) — характерное «хрипение» при тихих сигналах.

**Dithering** — добавление специального низкоуровневого шума, который **маскирует** quantization noise. Парадоксально, но добавление шума улучшает качество.

### Типы дизеринга

**TPDF (Triangular Probability Density Function):** стандартный, нейтральный.

**POW-R (Psychoacoustically Optimised Wordlength Reduction):** психоакустически оптимизированный — умнее маскирует шум.
- POW-R 1: для медленных, классических записей
- POW-R 2: для поп и рока
- POW-R 3: для яркой, высокочастотной музыки

### Когда применять dithering

**Применять:**
- При финальном экспорте в 16 bit
- Только **один раз** — в самом конце цепочки обработки

**НЕ применять:**
- В промежуточных экспортах (если работаете в 24 bit — сохраняйте в 24 bit)
- При экспорте 24 bit→24 bit (нет снижения bit depth)
- Дважды (двойное дизеринг ухудшает качество)

**Где найти дизеринг:**
- В настройках экспорта DAW (Ableton, Logic, Pro Tools)
- В мастеринг-плагинах (iZotope Ozone, Waves L3)
- Плагин UV22HR (Waves)

## Форматы файлов

### WAV (Waveform Audio File Format)

**Характеристики:**
- Lossless (без потерь)
- Большой размер (10 МБ/мин в 16bit/44.1kHz)
- Совместим со всеми платформами
- Поддерживает метаданные (BWF — Broadcast Wave Format)

**Применение:** студийная работа, архив, отправка дистрибьютору, CD.

**Качество для дистрибьюторов:** 16 bit / 44.1 кГц или 24 bit / 44.1 кГц (зависит от дистрибьютора).

### AIFF (Audio Interchange File Format)

**Характеристики:**
- Аналог WAV, разработан Apple
- Lossless, одинаковое качество с WAV
- Лучше поддерживает метаданные в Mac-среде

**Применение:** то же, что WAV — взаимозаменяемо.

### FLAC (Free Lossless Audio Codec)

**Характеристики:**
- Lossless сжатие (50–60% уменьшение размера без потерь)
- Открытый формат
- Отлично для архивирования и Hi-Res Audio

**Применение:** архив, Bandcamp, Tidal (lossless стриминг).

### MP3 (MPEG-1 Audio Layer 3)

**Характеристики:**
- Lossy сжатие — удаляет «незначимые» частоты
- Битрейт: 128, 192, 256, 320 kbps
- Маленький размер

| Битрейт | Качество | Применение |
|---------|---------|-----------|
| 128 kbps | Слышимая потеря качества | Не рекомендуется |
| 192 kbps | Приемлемо для большинства | Минимум для публикации |
| 256 kbps | Хорошее | iTunes Store |
| 320 kbps | Лучший MP3 | Spotify (premium) |

**Важно:** перед конвертацией в MP3 снизьте True Peak до **−2 dBTP** — MP3 кодирование может незначительно поднять уровень.

### AAC (Advanced Audio Coding)

Используется Apple Music, YouTube, Spotify на низких битрейтах. Качество лучше, чем MP3 при том же битрейте.

- Apple Music: AAC 256 kbps
- Spotify Free: AAC 128 kbps
- Spotify Premium: OGG Vorbis 320 kbps или AAC 256 kbps

## Рекомендуемый Workflow экспорта

### 1. Мастер-файл (архив)

```
Format: WAV или AIFF
Bit Depth: 24 bit
Sample Rate: исходный (44.1 или 48 кГц)
Dithering: НЕТ
```

Это ваш «оригинал» — сохраняйте навсегда.

### 2. Для дистрибьютора (Distrokid, TuneCore и др.)

```
Format: WAV
Bit Depth: 16 bit (или 24 bit — зависит от дистрибьютора)
Sample Rate: 44.1 кГц
Dithering: ДА (TPDF или POW-R)
True Peak: не выше −1.0 dBTP
```

### 3. Для видео

```
Format: WAV или AIFF
Bit Depth: 24 bit
Sample Rate: 48 кГц
Dithering: НЕТ (или 24→16 с dithering при необходимости)
```

### 4. Для YouTube / SoundCloud (быстро)

```
Format: MP3 или WAV
Bit Depth: 16 bit
Bitrate (MP3): 320 kbps
True Peak: −1.5 dBTP (с запасом для кодека)
```

## Проверка перед экспортом

- [ ] Мастер-шина не клиппирует
- [ ] LUFS соответствует целевой платформе
- [ ] True Peak не превышает −1.0 dBTP
- [ ] Sample Rate правильный (44.1 кГц для стриминга)
- [ ] Dithering включён (при экспорте в 16 bit)
- [ ] Метаданные заполнены (исполнитель, название, год)
- [ ] Файл назван правильно (Исполнитель - Название.wav)

> **Финальный совет:** всегда экспортируйте несколько форматов: 24bit WAV (архив), 16bit WAV (для дистрибьютора), MP3 320kbps (для демо и личного использования). Три файла — три разных сценария использования.
