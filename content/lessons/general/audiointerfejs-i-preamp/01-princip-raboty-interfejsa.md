# Принцип работы аудиоинтерфейса

## Зачем нужен аудиоинтерфейс

Компьютер умеет работать только с **цифровым сигналом** (последовательностью чисел). Микрофон и гитара вырабатывают **аналоговый сигнал** (непрерывно меняющееся напряжение). Аудиоинтерфейс — мост между этими двумя мирами.

```
Микрофон → [Преамп] → [ADC] → Компьютер → [DAC] → [Усилитель] → Мониторы
            (усиление)  (оцифровка)          (ЦАП)
```

## ADC — Аналого-Цифровой Преобразователь

**ADC (Analog-to-Digital Converter)** превращает непрерывный аналоговый сигнал в последовательность чисел.

Процесс называется **квантованием**: через равные промежутки времени (sample rate) ADC «снимает снимок» уровня аналогового сигнала и кодирует его числом (bit depth).

### Sample Rate (Частота дискретизации)

**Sample rate** — количество «снимков» сигнала в секунду.

| Sample Rate | Применение |
|------------|-----------|
| **44 100 Гц (44.1 кГц)** | CD-стандарт, стриминг, большинство музыки |
| **48 000 Гц (48 кГц)** | Видео и кино, broadcast |
| **88 200 / 96 000 Гц** | Высококачественная студийная запись |
| **176 400 / 192 000 Гц** | Архивное хранение, hi-res аудио |

По **теореме Найквиста**, sample rate должен быть минимум вдвое выше максимальной частоты, которую нужно записать. Человеческий слух — до ~20 кГц. Значит, 44.1 кГц теоретически достаточно.

**Зачем тогда 96 кГц?**
- Больше «пространства» при обработке (меньше артефактов при питче/варпинге)
- Запас при сведении и мастеринге
- Некоторые плагины работают лучше на высоком sample rate
- **Минус:** файлы в 2 раза больше, процессор нагружается сильнее

### Bit Depth (Разрядность)

**Bit depth** — сколько бит используется для кодирования каждого «снимка». Определяет **динамический диапазон**.

| Bit Depth | Динамический диапазон | Применение |
|----------|---------------------|-----------|
| **16 бит** | ~96 дБ | CD, стриминг |
| **24 бит** | ~144 дБ | Запись в студии (стандарт) |
| **32 бит float** | Практически неограничен | Внутренняя обработка DAW |

**24 бита** — стандарт для записи. 8 дополнительных дБ по сравнению с 16 битами дают огромный запас — можно писать тихо и не бояться обрезать тихие детали.

**32-bit float** — формат, используемый внутри DAW при обработке. Число может быть «за пределами» диапазона без клиппинга — DAW исправляет это при рендеринге.

## DAC — Цифро-Аналоговый Преобразователь

**DAC (Digital-to-Analog Converter)** делает обратное — превращает последовательность чисел обратно в аналоговое напряжение для усилителей и мониторов.

Качество DAC критично для **мониторинга** — то, что вы слышите в мониторах, определяется качеством ЦАП.

## Латентность (Latency)

**Латентность** — задержка между тем, что происходит (удар по клавише, щипок струны) и тем, что слышится в мониторах через интерфейс.

### Источники латентности

```
Микрофон → Интерфейс → Драйвер → DAW → Плагины → Драйвер → Интерфейс → Мониторы
            (ADC)     (buffer)         (обработка) (buffer) (DAC)
```

Каждый шаг добавляет задержку. Основная задержка — **buffer size** (размер буфера).

### Buffer Size

| Buffer Size | Латентность | Нагрузка на CPU |
|------------|-------------|----------------|
| 32–64 семпла | 1–3 мс (минимальная) | Очень высокая |
| 128–256 семплов | 3–6 мс (комфортная для записи) | Средняя |
| 512–1024 семпла | 10–25 мс (заметная задержка) | Низкая |
| 2048+ семплов | 40+ мс (мешает игре) | Минимальная |

**При записи** — уменьшайте buffer size (128–256 семплов).
**При сведении** — увеличивайте (1024–2048) — нагрузка меньше, плагины работают стабильнее.

## ASIO (Windows) и Core Audio (Mac)

Стандартные звуковые драйверы Windows (MME, DirectX) добавляют огромную латентность (100–300 мс). Для работы с аудио нужны специальные драйверы:

- **ASIO** (Audio Stream Input/Output, Windows) — стандарт для профессиональных интерфейсов
- **Core Audio** (macOS) — встроен в систему, хорошие задержки «из коробки»
- **ASIO4ALL** — бесплатный ASIO-драйвер для встроенных звуковых карт (работает, но хуже специализированного)

> **Практический вывод:** для серьёзной работы с аудио встроенная звуковая карта компьютера — не вариант. Даже бюджетный специализированный интерфейс (Focusrite Scarlett, Behringer UMC) даст несравнимо лучшую латентность и качество.
