# Мониторинг с нулевой латентностью

## Проблема латентности при записи

Когда вы записываете вокал или гитару, сигнал идёт:

```
Микрофон → Интерфейс (ADC) → Драйвер → DAW → Обработка → Драйвер → Интерфейс (DAC) → Наушники
```

Весь этот путь занимает **5–30 мс** (в зависимости от buffer size и системы). Это звучит как небольшая задержка, но для вокалиста или гитариста **даже 10 мс заметны** и мешают игре.

**Решение: мониторинг без прохождения через компьютер.**

## Direct Monitoring

**Direct Monitoring** — прямая подача входного сигнала на выход интерфейса, **до** обработки в компьютере.

```
Микрофон → Интерфейс (ADC) → [Direct Mix] → Интерфейс (DAC) → Наушники
                                    ↑
                            Аналоговый или цифровой микшер внутри интерфейса
                            (без задержки)
```

Задержка = время преобразования ADC/DAC ≈ **0.5–1 мс** — не воспринимается человеческим ухом.

### Ограничения Direct Monitoring

- Сигнал идёт **сухой** (без плагинов DAW)
- Нельзя добавить реверб/компрессию через DAW в реальном времени
- **Решения:** встроенные DSP-эффекты (UA Apollo), аппаратные эффекты или Hardware Insert

## DSP Mixing (Universal Audio Apollo)

Universal Audio Apollo решает проблему latency-free мониторинга с эффектами иначе:

1. Входной сигнал поступает в **DSP-процессор внутри Apollo**
2. DSP применяет UAD-плагины (реверб, компрессор, EQ) с **нулевой латентностью** (обработка на аппаратном уровне)
3. Обработанный сигнал идёт в наушники немедленно
4. Одновременно сухой (или обработанный — опционально) сигнал записывается в DAW

```
Микрофон → Apollo → [DSP плагины (LA-2A, 1073, реверб)] → Наушники (мгновенно)
                  → [Компьютер / DAW] → Запись
```

**Это означает:** вокалист слышит себя через Neve 1073 + LA-2A компрессор + Hall Reverb — **мгновенно**, без задержки.

### Console (программа Apollo)

**UA Console** — программный микшер для Apollo:
- Полный контроль над мониторингом
- До 64 каналов (при нескольких интерфейсах)
- Unison Technology — аппаратная имитация характера преампов (Neve, API, Helios)

## RME TotalMix FX

RME использует другой подход — **встроенный цифровой микшер TotalMix FX**:

- Работает независимо от DAW
- Поддерживает до 4096 каналов (в зависимости от модели)
- Встроенные EQ, компрессор, реверб (простые, не UAD-уровень)
- **Матрица маршрутизации** — любой вход на любой выход

TotalMix — стандарт для сложной маршрутизации в больших студиях.

## Buffer Size для записи

При записи с мониторингом через компьютер:

| Задача | Рекомендуемый Buffer Size |
|-------|--------------------------|
| Запись (максимальный комфорт) | 64–128 семплов |
| Запись (баланс) | 256 семплов |
| Сведение (много плагинов) | 512–1024 семплов |
| Мастеринг / рендеринг | 2048+ (не критично) |

**Практика:** используйте **direct monitoring при записи** (нулевая латентность) и **высокий buffer size** в DAW (для плагинов). Получаете лучшее из двух миров.

## Headphone Mix (Мониторинг для музыканта)

При записи в студии музыкант слышит **отдельный миксдаун** (cue mix) в наушниках — не тот, что слышит звукорежиссёр в мониторах.

**Типичный cue mix для вокалиста:**
- Себя (громко) + немного реверба (direct monitoring или DSP)
- Минус (фонограмма) — умеренно
- Барабаны — умеренно
- Остальные инструменты — тихо

Многие интерфейсы с 4+ каналами имеют **независимые выходы для наушников** с отдельным миксом.

## Latency Compensation в DAW

Когда вы используете разные пути (direct monitoring + запись), может возникать рассинхронизация между дорожками.

**Plugin Delay Compensation (PDC)** — функция DAW, автоматически компенсирующая задержку от плагинов. Включена по умолчанию в Pro Tools, Logic, Ableton Live, Reaper.

> **Итого:** для комфортной записи используйте direct monitoring (или DSP-мониторинг Apollo). Buffer size снижайте только настолько, насколько нужно для стабильной работы — малый buffer = высокая нагрузка на CPU. Найдите баланс: 128–256 семплов обычно достаточно.
