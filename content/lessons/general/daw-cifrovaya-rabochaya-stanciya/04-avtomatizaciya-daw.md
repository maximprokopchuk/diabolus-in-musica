# Автоматизация в DAW

## Что такое автоматизация

**Автоматизация** — запись изменений параметра во времени. Вместо того чтобы вручную двигать фейдер во время воспроизведения, вы рисуете кривую, которую DAW будет отрабатывать автоматически при каждом воспроизведении.

```
dB
  0 ┤         ╭──────────────╮
 -6 ┤        /               ╲
-12 ┤───────╯                 ╲────────
-18 ┤
    └──────────────────────────────── время
     [intro]  [куплет]  [припев]  [outro]
```

## Что можно автоматизировать

Практически **любой параметр** в DAW поддаётся автоматизации:

| Категория | Примеры |
|---------|--------|
| Трек | Громкость (volume), панорама (pan), mute |
| Плагины | Gain EQ, частота среза, время реверба, wet/dry |
| Синтезаторы | Cutoff, resonance, LFO rate, patch morph |
| Посылы | Уровень Send на ревербератор |
| Мастер | Финальная громкость для fade out |

## Режимы записи автоматизации

### Read (Воспроизведение)
DAW читает существующую автоматизацию и применяет её. Изменения в реальном времени не записываются.

### Write (Запись)
Во время воспроизведения любое движение параметра **немедленно записывается**, перезаписывая предыдущие данные. Используется для начального создания автоматизации.

### Touch
Запись начинается **только когда вы трогаете параметр** (например, берётесь за фейдер). Когда отпускаете — возвращается к предыдущей автоматизации. Идеально для точных правок.

### Latch
Как Touch, но когда отпускаете параметр, он **остаётся на последнем значении** и продолжает записывать. Удобно для поддержания нового уровня громкости.

### Trim / Relative
Не перезаписывает существующую автоматизацию, а **смещает её относительно** текущих значений. Полезно для общей корректировки уже записанной автоматизации.

## Рисование кривых вручную

Ручное редактирование автоматизации часто точнее и быстрее, чем запись в реальном времени.

### Типы кривых

**Линейная (Linear)** — равномерное изменение параметра. Используется для постепенных нарастаний.

**Кривая (Curve / Bezier)** — плавное изменение с ускорением или замедлением. Более естественно звучит для громкости (ухо воспринимает уровень логарифмически).

**Ступенчатая (Step)** — мгновенные скачки. Используется для ритмических эффектов (автоматизация mute в такт).

### Основные инструменты редактирования

| Инструмент | Действие |
|-----------|---------|
| Pencil (карандаш) | Нарисовать кривую от руки |
| Arrow (стрелка) | Выделить и двигать точки |
| Line (линия) | Нарисовать прямой отрезок |
| Curve | Согнуть прямой отрезок в кривую |
| Eraser | Удалить точки автоматизации |

## Автоматизация плагинов

Любой параметр вставленного плагина (EQ, компрессор, реверб) можно автоматизировать:

1. Откройте окно плагина
2. Правый клик на нужном параметре → «Show Automation» или «Add to Automation Lane»
3. DAW создаст отдельный трек/дорожку автоматизации для этого параметра

### Практические сценарии

**Автоматизация EQ для перехода:**
- В припеве добавить +3 dB на 3 кГц у вокала → голос «вылезает» вперёд
- На аутро срезать высокие частоты → эффект «удаления»

**Автоматизация компрессора:**
- Уменьшить Threshold в припеве → более плотный и напористый звук

**Автоматизация реверба:**
- Увеличить Wet Mix на окончаниях фраз → «дыхание» голоса

**Автоматизация фильтра:**
- Low-pass filter Sweep на синтезаторе → классический нарастающий эффект перед дропом

## Клип-автоматизация vs. Трековая автоматизация

| Тип | Описание | Когда использовать |
|----|---------|-----------------|
| **Track Automation** | Автоматизация на уровне трека, привязана к временной шкале | Изменения на протяжении всей песни |
| **Clip Automation** | Автоматизация внутри отдельного клипа | Повторяющийся паттерн (например, в каждом куплете) |

Clip-автоматизация «путешествует» вместе с клипом при перемещении — это удобно для повторяющихся секций.

## Советы по автоматизации

- **Fade in/out** начала и конца треков — обязательно. Предотвращает щелчки и клики.
- **Volume automation вместо фейдера** — оставьте фейдер в качестве «общего регулятора» и рисуйте детальную динамику на отдельном треке автоматизации.
- **Используйте маркеры** — расставьте метки (Verse, Chorus, Bridge) перед созданием автоматизации, чтобы ориентироваться в структуре.

> **Главное правило автоматизации:** она должна быть **неслышимой**. Хорошая автоматизация делает микс живым, но слушатель не замечает, что что-то меняется. Плохая автоматизация слышна как резкие скачки или неестественные переходы.
